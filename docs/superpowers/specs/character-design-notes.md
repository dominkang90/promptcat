## 비주얼 스타일

- **형태:** 통통하고 동글동글한(달걀/물방울 형태) 픽셀아트 고양이. 짧은 다리, 작은 세모 귀.
- **얼굴:** 점/짧은 선 눈, 작은 코, 가로선 수염. 표정은 단순하고 귀엽게.
- **외곽선:** 굵은 진갈색 픽셀 외곽선.
- **색감(대략):**
  - 외곽선: 진갈색 (#4a2f1a 계열)
  - 몸통: 따뜻한 회갈색/그레이지 (#b3a894 계열)
  - 줄무늬/얼룩: 진한 회갈색 (#8a7d6e 계열)
  - 하트 액센트: 분홍 (#f0a0b0 계열)
- **렌더링:** 픽셀아트(저해상도 격자). 안티앨리어싱 없이 또렷한 픽셀.

## 반응형 상태 ↔ 머리 위 이모트 매핑

캐릭터의 "반응형" 표현은 머리 위에 둥둥 뜨는 이모트 + 자세로 한다.
(아래는 제공된 3장에서 직접 도출 + 확장)

| 상태 | 이모트/자세 | 언제 |
|---|---|---|
| 대기(idle) | 🎵 음표, 늘어진 스플룻 자세 | 평소 가만히 있을 때 |
| 먹이 받음(fed) | 💗 하트 | 사진을 드래그/붙여넣기로 줬을 때 |
| 작업 중(working) | 🎵/⏳ (분석·생성 중) | 프롬프트 추출 또는 이미지 생성 진행 중 |
| 완료(success) | ✨ 반짝이(해) | 추출/생성 완료 |
| (추후) 오류 | 💢/❓ 등 | 실패 시 — 디자인 미정 |

> 이미지 생성(D/E) 중에도 캐릭터가 반응하도록 "working → success" 상태를 그대로 재사용한다.

## A 단계에서 정할 것 (지금은 미정)

- 스프라이트 시트 형식, 애니메이션 프레임 수
- 떠다니기/클릭 통과(click-through) 등 데스크톱 오버레이 동작
- 이모트 전환 트랜지션
