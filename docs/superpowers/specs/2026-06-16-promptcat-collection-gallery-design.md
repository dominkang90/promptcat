# C단계 — 컬렉션 갤러리 설계

작성일: 2026-06-16
상태: 승인됨 (구현 대기)

## 한 줄 요약

떠있는 고양이를 더블클릭하면 작은 로컬 웹 서버가 켜지고, 브라우저에 그동안 모은 모듈(추출 결과)들이 격자 갤러리로 열린다. 카드를 누르면 상세창에서 전체 프롬프트와 각 요소(고정/변동)를 복사할 수 있다.

## 배경 / 목적

지금은 사진을 고양이에게 먹이면 `modules/<유형>-<시간>/` 폴더에 `image`, `prompt.json`, `prompt.md`가 평평하게 쌓이기만 한다. 쌓인 것을 다시 **보고, 검색하고, 골라서 재사용(복사)** 할 방법이 없다. C단계는 이 "수집함을 둘러보고 골라 쓰는" 부분을 만든다.

이미지 생성(D)은 아직 없으므로, C에서 "재사용"은 **프롬프트(전체 또는 요소 단위)를 클립보드에 복사해서 다른 이미지 도구에 붙여넣기**를 의미한다.

## 범위 (이번에 만드는 것)

- 고양이 더블클릭(및 우클릭 메뉴)으로 갤러리 열기
- `modules/`를 읽어 갤러리 HTML을 그리는 순수 함수
- 갤러리와 사진을 서빙하는 작은 로컬 Node 서버 (외부 라이브러리 0개)
- 갤러리 화면: 검색 + 사진 카드 격자 + 카드 클릭 시 상세창
- 상세창에서 전체 프롬프트 / 고정요소 각각 / 변동요소 각각 복사

## 범위 밖 (이번엔 안 함)

- 이미지 생성(D), 테마 적용(E)
- 요소를 골라 새 프롬프트를 조립하는 기능
- 모듈 삭제·이름변경·태그·수동 하위폴더 만들기 (추후)
- 로그인/인증, 외부 공유

## 기술 선택

**작은 로컬 웹 서버 방식** (정적 `file://` 대신).

이유:
- 클립보드 복사 API는 "안전한 컨텍스트"에서만 동작 — `localhost`는 허용, `file://`은 브라우저에 따라 막힘.
- 서버가 열릴 때마다 `modules/`를 새로 읽어 항상 최신 상태를 보여줌.
- 사진도 서버가 함께 서빙해 경로 문제가 없음.
- Node 내장 `http`/`fs`만 사용 → 의존성 증가 없음.

## 구성 요소

작고 한 가지 일만 하는 단위로 나눈다.

### 1. `src/collection.ts` — 모듈 목록 읽기
- `listModules(baseDir: string): Promise<ModuleEntry[]>`
- 각 하위 폴더에서 `prompt.json`을 읽고 이미지 파일명을 찾아 항목으로 만든다.
- `ModuleEntry = { dir: string; imageFile: string; result: ExtractionResult }`
- `prompt.json`이 없거나 깨진 폴더는 건너뛴다(조용히 무시).
- 최신순(폴더명 역순) 정렬.

### 2. `src/gallery.ts` — 갤러리 HTML 생성 (순수 함수)
- `renderGallery(entries: ModuleEntry[]): string`
- 검색창 + 카드 격자 + 상세창(모달) + 클라이언트 JS(검색 필터, 모달 열기, 클립보드 복사)를 한 장의 HTML 문자열로 반환.
- 카드: 사진 썸네일(`/img/<dir>/<imageFile>`), 유형, 날짜.
- 상세 모달: 전체 프롬프트(복사 버튼), 고정요소 목록(각 value 복사), 변동요소 목록(각 value/placeholder 복사).
- 모듈 데이터는 페이지 안에 JSON으로 심어 두고 JS가 모달을 채운다.
- 사용자 텍스트는 HTML 이스케이프하여 깨짐/주입을 방지.

### 3. `src/gallery-server.ts` — 작은 로컬 서버
- 고정 포트(기본 4517). 라우트:
  - `GET /` → `listModules` 후 `renderGallery` 결과 반환
  - `GET /img/<dir>/<file>` → 해당 모듈 이미지 파일 반환 (경로는 `modules/` 안으로 제한)
- 진입점 실행 시 서버를 띄우고 주소를 stdout에 출력.

### 4. `scripts/open-gallery.sh` — 켜고 주소 알려주기
- 포트가 이미 떠있으면 그대로, 아니면 서버를 백그라운드로 켠다.
- 갤러리 주소(`http://localhost:4517`)를 stdout에 출력.

### 5. 고양이(`cat.pyw`)에 트리거 추가
- **더블클릭** → `open-gallery.sh`(WSL)를 호출해 주소를 받고 Windows 브라우저로 연다.
- 우클릭 메뉴에 **"📂 컬렉션 열기"** 추가.
- 기존 한 번 클릭/끌기(이동)·드롭(먹이기)·우클릭(먹이기/종료)은 그대로 둔다.

## 데이터 흐름

```
고양이 더블클릭
  → cat.pyw가 wsl로 open-gallery.sh 실행
  → 서버 켜짐(이미 떠있으면 재사용) → 주소 반환
  → cat.pyw가 그 주소를 Windows 브라우저로 엶
  → 서버가 modules/ 읽어 갤러리 HTML 그림
  → 사용자가 검색/카드 클릭/복사
```

## 에러 처리

- `modules/`가 없거나 비면: "아직 먹인 사진이 없어요 🐱" 안내 화면.
- 깨진/불완전한 모듈 폴더: 건너뜀.
- 이미지 라우트는 `modules/` 밖 경로 요청을 거부(경로 탈출 방지).
- 서버가 이미 떠있을 때 중복 실행: 포트 점검으로 한 개만 유지.

## 테스트 전략 (TDD)

- `collection.ts`: 임시 폴더에 가짜 모듈을 만들어 `listModules`가 올바른 항목/정렬/건너뜀을 내는지.
- `gallery.ts`: `renderGallery`가 유형·전체프롬프트·요소 value와 이미지 경로(`/img/...`)·복사용 데이터를 HTML에 담는지. HTML 이스케이프 동작.
- `gallery-server.ts`: 가벼운 실행 확인(`/`가 200과 HTML, 잘못된 이미지 경로는 거부).

## 성공 기준

- 고양이를 더블클릭하면 브라우저에 갤러리가 열린다.
- 먹인 모듈들이 사진 카드로 보이고, 검색으로 걸러진다.
- 카드를 누르면 상세창에서 전체 프롬프트와 각 요소를 복사할 수 있고, 다른 곳에 붙여넣기가 된다.
- 새 사진을 먹인 뒤 갤러리를 다시 열면 바로 나타난다.
- 단위 테스트 통과 + 타입검사 통과.
